## v2 in one paragraph

Three branches (Creativity `#FF3333`, AI Skills `#FFCC00`, Web & App Design `#33CCFF`) grow
from a shared white root, reproducing the spec's approved Mermaid topology node-for-node,
including the 11 unlabeled decorative connectors. Modules unlock sequentially per arm on
click (blocked clicks shake + hint), state persists in localStorage, counters track branch
and total progress, tooltips carry the spec's descriptions verbatim, keyboard + reduced
motion supported, and under 768px the tree becomes three stacked branch lists. Everything is
one self-contained `index.html` (fonts base64-inlined; Chakra Petch display, Inter body).

## Sources of truth

- **Behavior + topology + tokens:** the v2 build spec (path above). Its Mermaid diagram is
  authoritative for structure; §3's table is authoritative for names/descriptions (with the
  §5 display-name mapping, e.g. "Stay Curious" renders as "Curiosity").

## Verification

`npm run verify` (Playwright) covers data sync, node/edge counts vs the Mermaid graph,
all 21 tooltips verbatim, sequential unlock + cascade re-lock, persistence, keyboard,
reduced motion, mobile stack, and console errors. `node scripts/visual-crops.mjs` writes
high-res branch crops to `verify-out/` for visual review.

## Deliberate decisions (don't relitigate casually)

- **Single self-contained file, no runtime network** — carried over from v1; also keeps the
  page portable to any static host.
- **Deploy stays GitHub Pages** (push to `main` → live). The spec mentions Vercel; Pages was
  already live and serves the same "shareable URL" purpose. Moving to Vercel is a
  15-minute task if ever wanted.
- **Re-locking cascades** to dependents so the sequential invariant can never break.
- **"Available" modules** (parent unlocked) show a branch-color ring so the next affordance
  is visible; fully locked modules stay gray per the spec's locked tokens.
